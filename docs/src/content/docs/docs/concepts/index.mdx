Keelson is the local harness around coding agents. It does not replace the
agent; it gives the agent a governed place to run, remember, repeat work, call
tools, and surface results. The concept pages build that mental model: what
keelson is, why it is shaped this way, and the vocabulary you will meet
everywhere else.

Each page carries one organizing idea, and they read best in order: the
positioning first, then the map, the live chat loop, the extension model, the
two engines, where their work is scoped, the state behind them, and the provider
underneath every turn.
