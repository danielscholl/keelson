The tutorials are one rail, not a menu. Each page opens with the problem the
last one left you holding, ends at a working checkpoint, and hands off to the
next. Every command and its output is real, and the first three pages run
without credentials: the offline stub provider stands in wherever a real
coding agent would, so you can verify the harness before you bring keys.

Ride it in order. The capstone, building your own rib, is what the first two
pages are quietly preparing you for, and the closing field exercise brings real
models to everything you learned.

The whole rail is four stops, and you can ride the first three before you bring
any credentials:
