Keelson is a single-user agent harness. You chat with a coding agent, run
repeatable agent chores as deterministic YAML workflows, and keep every
conversation, run, and secret on your own machine, with no Keelson-hosted
service in the loop. The harness is the keel, the part that ships and is useful
on its own;
ribs are the capabilities you bolt on for more.

These docs are tiered by audience. Start with Concepts to learn the shape of the
system, follow the Tutorials to build, and reach for the Reference when you need
the contract.
